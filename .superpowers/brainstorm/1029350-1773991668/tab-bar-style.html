Tab Bar Style

How should the tab bar look and feel in Ghostty for Windows?

A

Custom-drawn (like GTK Adwaita)

Render the tab bar ourselves using GDI/Direct2D. Full control over appearance — can match Ghostty's dark theme, use the same tab styling as GTK (rounded tabs, close buttons, new-tab button). This is what most modern terminal emulators do (Windows Terminal, WezTerm).

Pros

  • Full visual control — matches Ghostty branding
  • Can implement all GTK tab features (wide tabs, tab overview, drag reorder)
  • Consistent look across themes/Windows versions
  • Windows Terminal and WezTerm both do this

Cons

  • More code to write and maintain
  • Must handle hit-testing, hover, drag-drop ourselves
  • Accessibility needs manual work (screen readers)
B

Win32 Tab Control (native)

Use the built-in Win32 WC_TABCONTROL common control. Looks like a native Windows property sheet with rectangular tabs. Functional but dated appearance.

Pros

  • Very little code — Windows handles everything
  • Built-in accessibility/screen reader support
  • Native Windows look

Cons

  • Looks like a 1990s Windows dialog — dated and ugly
  • Very limited styling — can't match Ghostty's dark theme
  • No close button per tab, no drag reorder
  • No modern terminal does this
C

Hybrid: Simple custom bar now, enhance later

Start with a minimal custom-drawn tab bar — just tab titles, active indicator, close button, new-tab button. Skip drag reorder and tab overview for v1. Iterate towards full feature parity later.

Pros

  • Gets real tabs working quickly
  • Custom drawing gives us the dark theme match
  • Iterative — ship basic tabs, add polish later
  • Less code than full custom solution upfront

Cons

  • Still need custom drawing code (but simpler)
  • Missing features initially (drag reorder, overview)
