Tab Bar Layout & Position

Where should the tab bar sit and how should it interact with the title bar?

A

Below title bar (standard)

Keep the native Windows title bar (with min/max/close buttons). Tab bar sits directly below it, above the terminal area. Similar to Chrome's original design or VS Code's tab layout.

Simpler to implement. Native title bar handles window dragging, min/max/close. Tab bar is just a child region.

B

Integrated into title bar (like Windows Terminal)

Tabs live inside the title bar area itself. Requires extending into the non-client area (DwmExtendFrameIntoClientArea). Saves vertical space. This is what Windows Terminal, Chrome, and Edge do.

More polished look, saves a row of pixels. But requires DWM non-client area customization — significantly more complex (hit testing, caption buttons, DPI handling).

C

Option A first, Option B later

Ship with tabs below the title bar initially. Add title bar integration as a follow-up (could be a config option like GTK's gtk-titlebar-style: tabs vs native).

This mirrors GTK which supports both modes via config. Start with the simpler "native" style, add the integrated "tabs" style as an enhancement.
